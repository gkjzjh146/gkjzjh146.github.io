# 参考链接 [数据库取证分析 | Getuid (doomkkkkk.github.io)](https://doomkkkkk.github.io/2021/10/23/数据库取证分析/) https://mp.weixin.qq.com/s/vNwtQg6JpUBnnORNTx1BOA # MYSQL 搭建环境(mysql8) ```shell docker pull mysql docker run -d --name mysql -e MYSQL_ROOT_PASSWORD=password mysql mysql -h 127.0.0.1 -uroot -ppassword ``` 搭建环境mysql(5.7) ```shell docker pull mysql:5.7 docker run -d --name mysql5.7 -e MYSQL_ROOT_PASSWORD=password mysql:5.7 mysql -h 127.0.0.1 -uroot -ppassword ``` ## MYSQL简介 ### mysql文件： - .ini/cnf - .err错误文件 - ib_logfile事务日志文件（innodb） - binlog二进制日志文件 - .frm表结构文件 - .ibdata共享表空间文件（innodb） - .ibd独立表空间文件（innodb） - .myi索引文件（myisam） - .myd数据文件（myisam） ### 配置文件 **windows**：my.ini或者my-default.ini **linux**：/etc/my.cnf ``` [mysqld] # 设置3306端口 port=3306 # 设置mysql的安装目录 basedir=D:\Program Files\MySQL # 设置mysql数据库的数据的存放目录 datadir=D:\Users\qiuzhiwen\MySQL\Data # 允许最大连接数 max_connections=200 # 允许连接失败的次数。这是为了防止有人从该主机试图攻击数据库系统 max_connect_errors=10 # 服务端使用的字符集默认为UTF8 character-set-server=utf8 # 创建新表时将使用的默认存储引擎 default-storage-engine=INNODB # 默认使用“mysql_native_password”插件认证 default_authentication_plugin=mysql_native_password [mysql] # 设置mysql客户端默认字符集 default-character-set=utf8 [client] # 设置mysql客户端连接服务端时默认使用的端口 port=3306 default-character-set=utf8 ``` ### 日志文件 登录mysql后可以通过sql语句进行查看 **日志文件路径** ``` show variables like 'general_log_file'; ``` **错误日志文件路径** ``` show variables like 'log_error'; ``` **慢查询日志文件路径** ``` show variables like 'slow_query_log_file'; ``` ### MYsql结构： > data目录： > > - 如果数据库用的是myisam引擎，则目录底下有 > - .frm(表文件) > - .myd(数据) > - .myi(索引)文件 > - 如果数据库用的是innoDB引擎，则目录下有 > - .frm(表文件) > - .ibd(数据)文件 ## 版本识别 ### 登录 登录成功后可以直接看到版本信息 ![image-20230902183903286](/数据库取证.assets/image-20230902183903286.png) ### status ``` mysql -h 127.0.0.1 -uroot -ppassword status; ``` ![image-20230902184140767](/数据库取证.assets/image-20230902184140767.png) ### select version(); ``` select version(); ``` ![image-20230902185015370](/数据库取证.assets/image-20230902185015370.png) ### mysql -v ``` mysql -v ``` ### 第三方工具查看 用宝塔查看 ![image-20230902185943717](/数据库取证.assets/image-20230902185943717.png) ### 十六进制查看 用010editor打开数据库文件夹下任意后缀为frm的文件，版本偏移量为 34+33（小端） 的值，如图所示是5.7.26的版本。 ![image-20230903221849502](/数据库取证.assets/image-20230903221849502.png) ## 数据库存储路径 - windows下数据文件路径：C:\programdata\MYSQL\Mysql sever 5.7.29\data - linux下数据文件路径：/var/lib/mysql/ 仿真起来可以使用 ``` show global variables like "%datadir"； ``` ![image-20230903223017555](/数据库取证.assets/image-20230903223017555.png) 还可以利用文件的扩展名来查找数据存储路径， ``` .mdf .ldf .bak .sql ``` ## 绕过数据库密码 ### 操作 ``` 1.停止MySQL服务。 systemctl stop mysqld 2.找到MySQL的配置文件所在路径。 find / -name my.cnf 3.修改配置文件。 vim /文件所在路径/my.cnf 4.在配置文件中【mysqld】后面加入语句。 skip-grant-tables 5.开启MySQL服务。 systemctl start mysqld 6.免密登录数据库。 mysql -uroot -p（此处按两次回车即可进入） ``` ### 例子：2023蓝帽杯服务器取证 32 ![image-20230904001314519](/数据库取证.assets/image-20230904001314519.png) ## 修改root密码： 取证一般能拿到root密码，但是如果没能拿到root密码，可以用下面方法来改密码,来用来连接navicat ``` mysql -uuser -ppassword ``` **方式一：** ``` ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '123456'; ``` **方式二：** ``` show databases; use mysql; ``` （这里的mybase是数据库名） 5.7.1以下版本 ``` update user set password=password('p4ssword') where user='root'; ``` 5.7.1以上版本 ``` update user set authentication_string=password('p4ssword') where user='root'; flush privileges; quit ``` ![image-20230904120920021](/数据库取证.assets/image-20230904120920021.png) ## 图形化处理 使用工具navicat ### 连接方式（常规） 这里可以先看数据库的ip用`ifconfig`然后输入账号密码直接连接 ![image-20230904002332774](/数据库取证.assets/image-20230904002332774.png) ### 连接方式（ssh） ![image-20230904002614425](/数据库取证.assets/image-20230904002614425.png) 如果是没开3306端口，可能是使用ssh通道进行连接 ### 数据恢复 如果是db文件，可以直接将其拖到navicat ### 题目：2023蓝帽杯服务器取证 ![image-20230904002909893](/数据库取证.assets/image-20230904002909893.png) # Oracle: 取证中不常见，先占坑 # Mongodb ## 环境搭建 ``` docker pull mongo docker run -d --name my-mongodb -p 27017:27017 mongo ``` 可以用navicat直接连接 ![image-20230904165405639](/数据库取证.assets/image-20230904165405639.png) ## 简介 MongoDB 是一个基于分布式文件存储的数据库。由 C++ 语言编写。旨在为 WEB 应用提供可扩展的高性能数据存储解决方案。MongoDB 是一个介于关系数据库和非关系数据库之间的产品，是非关系数据库当中功能最丰富，最像关系数据库的 ## 重要文件 ### 配置文件 **linux:** ``` /etc/mongodb.conf ``` **windows:** 这个得看具体情况找 ``` mongodb.config ``` ### mongodb.conf ``` # mongodb.conf # 数据库文件位置 dbpath=/var/lib/mongodb #日志文件的路径 logpath=/var/log/mongodb/mongodb.log # 是否追加方式写入日志，默认True logappend=true # 设置绑定ip bind_ip = 127.0.0.1 # 设置端口 port = 27017 # 是否以守护进程方式运行，默认false fork = true # 启用日志文件，默认启用 journal=true # 启用定期记录CPU利用率和 I/O 等待,默认false #cpu = true # 是否以安全认证方式运行，默认是不认证的非安全方式 #noauth = true #auth = true # 详细记录输出，默认false #verbose = true #用于开发驱动程序时验证客户端请求 #objcheck = true # # 启用数据库配额管理,默认false #quota = true # 设置oplog日志记录等级，默认0 # 0=off (default) # 1=W # 2=R # 3=both # 7=W+some reads #oplog = 0 # 是否打开动态调试项，默认false #nocursors = true # 忽略查询提示，默认false #nohints = true # 禁用http界面，默认为localhost：28017 #nohttpinterface = true # 关闭服务器端脚本，这将极大的限制功能，默认false #noscripting = true # 关闭扫描表，任何查询将会是扫描失败 #notablescan = true # 关闭数据文件预分配 #noprealloc = true # 为新数据库指定.ns文件的大小，单位:MB # nssize = # 用于Mongo监控服务器的Accout token。 #mms-token = # Mongo监控服务器的服务器名称。 #mms-name = # Mongo监控服务器的Ping间隔时间，即心跳 #mms-interval = # Replication Options # 设置主从复制参数 #slave = true # 设置从节点 #source = master.example.com # 指定从节点的主节点 # Slave only: 指定要复制的单个数据库 #only = master.example.com # or #master = true # 设置主节点 #source = slave.example.com # 设置副本集的名字，所有的实例指定相同的名字属于一个副本集 replSet = name #pairwith = # 仲裁服务器地址 #arbiter = # 默认为false，用于从实例设置。是否自动重新同步 #autoresync = true # 指定的复制操作日志（OPLOG）的最大大小 #oplogSize = # 限制复制操作的内存使用 #opIdMem = # 设置ssl认证 # Enable SSL on normal ports #sslOnNormalPorts = true # SSL Key file and password #sslPEMKeyFile = /etc/ssl/mongodb.pem #sslPEMKeyPassword = pass ``` ### 日志文件 具体位置还得先看配置文件，下面的是默认的 ``` /var/log/mongodb/mongodb.log ``` ## 基本操作 **连接数据库**： ``` mongodb://[username:password@]host1[:port1][,host2[:port2],...[,hostN[:portN]]][/[database][?options]] ``` 一般就是看账号密码，端口默认是27017，如果填了，就按照填写的来 使用用户名gkjzjh146，密码password登录localhost的database数据库 ``` mongodb://gkjzjh146:password@localhost/database ``` **恢复数据库** 可以提前准备好docker，直接用docker来搭建，将数据库文件拖进navicat即可 ##
